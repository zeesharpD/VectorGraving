Vector Graving SVG Music Notation

Even though there is no Graphical User Interface for this process, the advantages are considerable over a music notation program. It is 100% extensible. There is nothing you cannot have in you're files, or how the symbols are placed. Every existing glyph, symbol, item in the library can be altered to suit yourself. All of the most complex symbols are already in the library. It only takes me a matter of minutes to create the ones that aren't there yet, and I will gladly provide them for you at you're request, while also showing you how easy it is to do you're self.

Viewing the files/templates is no longer a matter of choice. Modern browsers can read 'stand alone' SVG files, but these files are linked to the libraries; an attribute removed from the latest version of the SVG(2) specification -without explanation-, and all the browser developers have complied with it's removal! Apache's Batik application remains unbroken. It still recognizes SVG 1.1. You will need to install it for both *viewing and converting to PDF. You will also need 'Java' installed for Batik, if you don't already have it.

* Having a browser also is very convenient. I'm using an old version of Firefox (v.10.0), just for typesetting music.

If you don't already have one, I highly recommend getting an advanced text/code editor. There are many available: Editor Index. Programs such as these make editing code much easier, and the files you will be creating are, after all, text files.

As a point of clarity, there are many places in the help files that show +x or +y. These are for emphasis, not what is actually written in the code. Only negative numbers require the operator '-', a hyphen, not an actual minus '−' sign. The upper left-hand corner of all blocks or groups is 0,0. Right is positive x. Down is positive y. Left is negative x. Up is negative y.

After reviewing an index, the primer, and template usage, open a title page template in both a viewer and a text editor. If you just want to experiment, putting in a few items will not ruin the template. If you intend to keep going, 'save as' and the new name you choose. Make whatever changes you like as pointed-out in 'TemplateUse.htm'. Replace the words 'Title', 'sub title', and 'Composer', with the actual names. If there is no sub title, delete that line.
